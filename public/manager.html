最初の使い方

Manager に作業を起票する流れと、下からそのまま送る相談を分けて使えます

通常の相談は下の送信欄からそのまま送れます。isolated worktree で進めたいものだけ「新しい作業」から起票すると、Manager が既存 repo か新規 repo かを判断して進めます。

どう整理されるかを見る

「新しい作業」から起票したものは、Manager が既存 repo か新規 repo かを判断したうえで isolated worktree に回します。相談として送った内容は、既存の作業項目への追記、新しい作業項目、確認が必要なものに分かれます。

1. 「新しい作業」にタイトルと依頼を書く
2. mode を決めて起票し、repo 判断は Manager に任せる
3. queued / working / review をこの一覧で追い、あとで merge lane へつなげる
